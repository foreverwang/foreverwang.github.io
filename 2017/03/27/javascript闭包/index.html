Javascript闭包

发表于 2017-03-27 | 分类于 javascript |

javascript蜜汁闭包

说起闭包，对很多人大概是一个这样的状态：我知道这是一个闭包，但却说不出一个闭包是什么。犹如一个欲拒还迎半遮半掩的姑娘。

看定义，真是含糊不清；看各种文章又总是片面浅显。 这里我试图一层一层剥开闭包的外衣，还原一个赤裸裸的闭包。本文结束后你会发现，闭包无处不在；我们根本不需知道闭包的具体定义是什么。

什么是闭包？

先看下定义：

闭包是指可以包含自由（未绑定到特定对象）变量的代码块；这些变量不是在这个代码块内或者任何全局上下文中定义的，而是在定义代码块的环境中定义（局部变量）。

单看这个定义好像还是不能知道javascript中闭包是什么。

那闭包在javascript是怎么实现的呢？

答案是：词法作用域+作用域链。

如果你已经彻底搞懂了javascript执行上下文，那么你已经了解了闭包的本质了，具体闭包的定义是什么已经完全不重要了。

ECMA262中规定JS使用Scope Chain来实现闭包。
js采用词法作用域，函数的执行依赖于变量作用域，这个作用域是在函数定义时决定的（而不是函数决定时调用的）。函数对象的内部状态不仅包含函数体内的代码逻辑，还必须引用当前的作用域链。函数体内部的变量可以保存在函数作用域内，这种特性叫做闭包。

作用域链 = VO + [[scope]]

[[scope]]是函数的一个内部属性，当函数定义的时候就已经确定，这个属性的值是由父执行环境的变量对象组成的一个链表。

当一个函数执行的时候，函数内标志符的解析就是在作用域链里查找的，先在VO里查找，若找不到就从[[scope]]内的下一个变量对象继续查找。
这使得函数能记住定义它时的词法作用域，不管这个函数在那里调用。这就是闭包。
